zookeeper之ExpiryQueue

zookeeper之ExpiryQueue

ExpiryQueue是在SessionTrackerImpl中使用的，主要是维护zk中session的队列
了解ExpiryQueue的实现

//ExpiryQueue<SessionImpl> sessionExpiryQueue;

//elemMap保存的是SessionImpl-->过期时间
private final ConcurrentHashMap<E, Long> elemMap = new ConcurrentHashMap<E, Long>();

//expiryMap保存的是 过期时间-->SessionImpl集合
private final ConcurrentHashMap<Long, Set<E>> expiryMap = new ConcurrentHashMap<Long, Set<E>>();

****elemMap和expiryMap是利用空间换时间，方便根据SessionImpl 和 过期时间相互查找****

ExpiryQueue的remove方法

ExpiryQueue的实现相当于一个缓存，想起之前面试的一个题目：实现一个缓存fifo,缓存的value可以有过期时间/无过
期时间，缓存大小一定，当put时，如果缓存满了(达到缓存上线)，优先剔除最近要失效的key-value，如果key都不过
期，优先剔除最先进来的key，直到能放下新的key-value。这个题可以用ConcurrentHashMap缓存，TreeSet(有序集
合)过期时间排序，或者二分查找。
**从这里感觉算法的重要性。时间复杂度、空间复杂度在实际工作中得以应用。**

public Long remove(E elem) {
     //快速根据SessionImpl获取过期的时间
    Long expiryTime = elemMap.remove(elem);
    if (expiryTime != null) {
        Set<E> set = expiryMap.get(expiryTime);
        if (set != null) {
                //从expiryMap中删除
            set.remove(elem);
        }
    }
    return expiryTime;
}

ExpiryQueue的update方法

public Long update(E elem, int timeout) {
    Long prevExpiryTime = elemMap.get(elem);
    long now = Time.currentElapsedTime();
    Long newExpiryTime = roundToNextInterval(now + timeout);

    if (newExpiryTime.equals(prevExpiryTime)) {
        // No change, so nothing to update
        return null;
    }

    // First add the elem to the new expiry time bucket in expiryMap.
    Set<E> set = expiryMap.get(newExpiryTime);
    //**如果elemMap中不存在，放入到elemMap中
    if (set == null) {
        // Construct a ConcurrentHashSet using a ConcurrentHashMap
        set = Collections.newSetFromMap(
            new ConcurrentHashMap<E, Boolean>());
        // Put the new set in the map, but only if another thread
        // hasn't beaten us to it
        Set<E> existingSet = expiryMap.putIfAbsent(newExpiryTime, set);
        if (existingSet != null) {
            set = existingSet;
        }
    }
    set.add(elem);

    // Map the elem to the new expiry time. If a different previous
    // mapping was present, clean up the previous expiry bucket.
    prevExpiryTime = elemMap.put(elem, newExpiryTime);
    if (prevExpiryTime != null && !newExpiryTime.equals(prevExpiryTime)) {
        Set<E> prevSet = expiryMap.get(prevExpiryTime);
        if (prevSet != null) {
            prevSet.remove(elem);
        }
    }
    return newExpiryTime;
}
